# 3. Detailed Specifications

## 시스템 통합 대시보드
- [목적] 전체 회선과 유닛의 상태를 한눈에 파악하고 명령을 하달하는 중앙 관제 센터.
- [기능] 
    - 활성 회선(Active Circuit) 표시 및 전환 인터페이스.
    - 실시간 오토파일럿 파이프라인 진행 상태 (Step 1~7) 시각화.
    - 통합 로그 및 감사 보고서(Audit Report) 실시간 스트리밍.

## 센티널 오토파일럿 엔진
- [목적] 지휘관의 의도를 기술적 실체로 변환하고 전 과정을 자율 제어하는 핵심 브레인.
- [특징]
    - 능동적 적응형 오케스트레이션: 실시간 Git 맥락 수집을 통한 전술 수립.
    - 무관용 감사 원칙: 규약 위반 시 물리적 차단 및 피드백 제공.
    - 학습 루프: 미션 완수 시 전술적 지식을 영구 저장하여 다음 미션에 반영.

## 다국어 표준 파서 유닛
- [목적] 전사 데이터의 다국어 무결성 보장을 위한 표준 기술 유닛.
- [기능]
    - Pydantic 기반의 엄격한 데이터 검증 및 누락 필드 자동 태깅.
    - ko/en 필드 필수 보정 로직을 통한 UI 정합성 확보.

## MVVM 기반 UI 유닛
- [목적] 정보 손실 없는 High Signal UI 구현을 위한 Swift 전문 유닛.
- [특징]
    - Dummy View 원칙 준수: ViewController는 오직 이벤트 전달만 담당.
    - 관심사 분리: `createInput()` 메서드를 통한 등록과 생성 로직 격리.

## 워크플로우 기획 유닛
- [목적] 기획 단계부터 구조적 결함이 없는 문서를 작성하는 전략 유닛.
- [특징]
    - User Workflow 선언 필수화.
    - TOC(목차)와 상세 기획 헤더의 100% 일치성 강제.
    - 예외 케이스 및 정책 명시 의무화.
