# [ADR] 중앙 집중식 I18N 파서 도입 및 데이터 검증 규격

## 1. 배경 (Context)
호버크래프트 UI는 글로벌 서비스를 위해 다국어를 지원해야 하나, 기존 방식은 각 컴포넌트나 모듈에서 개별적으로 언어 필드를 처리하고 있어 다음과 같은 문제가 발생함:
- 언어 필드(`ko`, `en`) 누락 시 런타임 에러 발생 가능성.
- 다국어 데이터 구조의 불일치로 인한 파싱 로직 중복.
- 전체 데이터의 I18N 준수 여부를 파악하기 어려움.

## 2. 결정 (Decision)
1. **중앙 파서 구현:** `mcp_operator` 엔진 내에 표준 I18N 파서를 구현하여 모든 다국어 필드 접근을 일원화한다.
2. **엄격한 타입 체크:** Pydantic 모델을 사용하여 다국어 객체의 구조를 강제한다.
3. **자동 폴백(Fallback) 메커니즘:** 특정 언어 필드 누락 시 로그를 생성하고 사전에 정의된 기본 언어로 자동 전환한다.
4. **Sentinel Audit 연동:** `SENTINEL` 유닛의 감사 단계에 I18N 정합성 체크를 필수 항목으로 포함한다.

## 3. 결과 (Consequences)
- **장점:** 코드 중복 제거, 데이터 안정성 확보, 전사적 다국어 준수율 가시화.
- **단점:** 초기 데이터 마이그레이션 비용 발생, 파서를 통한 데이터 접근으로 인한 미세한 오버헤드 (캐싱으로 보완 예정).

## 4. 트레이드오프 (Trade-offs)
- 유연한 스키마 대신 엄격한 규격을 선택함으로써 개발 초기 속도는 다소 느려질 수 있으나, 장기적인 유지보수 및 런타임 안정성을 확보함.
